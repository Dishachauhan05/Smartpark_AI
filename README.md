**SmartPark AI** is a professional gate registration portal and real-time stay telemetry console. Built for modern commercial hubs and residential zones, it enables drivers to check in using their mobile devices, logs entries securely, and generates operational analytics utilizing Google Gemini AI.

---

## 🌟 Core Features

- 🖥️ **Central Command Dashboard**: Live occupancy counters, active stay stats, and searchable logs.
- 📊 **Dynamic Stay Analytics**: Dynamic calculations of average parking duration and peak traffic check-in hours.
- 📝 **Fullscreen Kiosk Entry**: Mobile-optimized registration flow for guests at gate check-in panels.
- 🎟️ **Digital Pass Receipt**: Generates check-in receipts with animated laser barcode strips and ticket details.
- 🔍 **License Plate Lookup**: Database search interface displaying visitor profile summaries and checkout controls.
- 🔮 **Gemini AI Insights**: High-performance operational analysis reports with fallback local engines.
- 📱 **QR gate Pass Board**: Printable gate flyer with origin-aware scanning URLs for mobile testing.

## 🚀 Getting Started

### 1. Clone & Install
```bash
git clone https://github.com/your-username/smartpark-ai.git
cd smartpark-ai
npm install
```

### 2. Configure Environment Keys
Create a `.env.local` file in the root directory:
```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anonymous-public-key
GEMINI_API_KEY=your-google-gemini-api-key
```

### 3. Initialize Postgres Tables (Supabase SQL Editor)
Execute this SQL script inside your Supabase project query terminal to create the required relational schemas:

```sql
-- Create Visitors Directory Table
CREATE TABLE visitors (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  purpose TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Create Vehicles Stay Log Table
CREATE TABLE vehicles (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  visitor_id BIGINT REFERENCES visitors(id) ON DELETE CASCADE,
  vehicle_number TEXT NOT NULL,
  vehicle_type TEXT NOT NULL,
  status TEXT DEFAULT 'ACTIVE' NOT NULL,
  exited BOOLEAN DEFAULT FALSE NOT NULL,
  entry_time TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  exit_time TIMESTAMP WITH TIME ZONE
);
```

### 4. Run Development Node
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) on your local browser to view the gateway portal.

## 📱 Mobile Kiosk Testing Guide

Since the guest registration form is accessed by visitors scanning a QR code with their mobile devices:
1. Connect both your host computer and test mobile phone to the **same Wi-Fi network**.
2. Run your Next.js server locally on port 3000.
3. Access the dashboard on your computer using your local network IP (e.g., `http://192.168.x.x:3000/qr`).
4. The QR code generator will dynamically encode the correct IP address in the scan value, allowing your mobile phone to check in successfully!
